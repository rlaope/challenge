# 챌린지 (Challenge)

이 저장소는 제너럴리스트(general-purpose) 엔지니어를 위한 두 개의 테이크홈(take-home) 챌린지를 담고 있습니다.

목표는 사전(事前) 도메인 전문성을 평가하는 것이 **아닙니다**. 목표는 지원자가 **낯선 문제 영역에 진입하여**, 올바른 질문을 던지고, 효과적으로 검색하며, 핵심 정보를 찾아내고, AI를 잘 활용하여, 이를 **실용적인 해결책**으로 전환할 수 있는지를 평가하는 것입니다.

## 평가 기준 (What We Evaluate)

- **학습 능력 (Learning):** 지원자가 낯선 도메인에 얼마나 빠르게 진입하여 효과적으로 일할 수 있는가.
- **검색 능력 (Search):** 얼마나 잘 리서치하고, 올바른 질문을 던지며, 신호와 잡음(signal vs noise)을 분리해 내는가.
- **문제 해결 (Problem Solving):** 불완전한 정보를 어떻게 실행 가능한 계획이나 작동하는 솔루션으로 전환하는가.
- **판단력 (Judgment):** 일반론적인 답변 대신 **가장 레버리지가 높은(highest-leverage) 변경점**을 찾아내는가.
- **범위 (Range):** 리서치 중심의 아키텍처 작업과 직접 손을 더럽히는 리버스 엔지니어링을 모두 다룰 수 있는가.
- **커뮤니케이션 (Communication):** 추론 과정, 트레이드오프, 검증 방법을 얼마나 명확하게 설명하는가.
- **AI 활용 (AI Usage):** AI를 탐색·학습·실행 도구로서 얼마나 효과적으로 활용하는가.

## 챌린지 (Challenges)

### 1. [FLUX.2 추론 최적화 챌린지](./task-ai.ko.md) (4시간)

![](../assets/ai2.png)

낯선 ML 추론 시스템을 학습하여, 이 모델·이 유스케이스에 특화된 **아키텍처 인지(architecture-aware) 지연 시간 최적화**를 제안하는 과제. 지원자가 얼마나 빠르게 학습·리서치하고, 가장 레버리지가 높은 변경점을 찾아내는지를 본다.

### 2. [한컴 닥스(Hancom Docs) 자동화 SDK 챌린지](./task-web.ko.md) (4시간)

![](../assets/web1.svg)

공개 API가 없는 **캔버스(`<canvas>`) 기반 에디터**를 리버스 엔지니어링하여, **CDP(Chrome DevTools Protocol) 기반 SDK**를 구축하는 과제. 불투명한(opaque) 시스템을 얼마나 잘 탐색하여 핵심 인사이트를 찾고, 작동하는 소프트웨어로 만들어내는지를 본다.
